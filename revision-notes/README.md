### Naming Convention

**Format**: `{X.Y}-L{depth}-{semantic-concept}.md`

- **{X.Y}**: Lesson number (e.g., 3.1, 3.17)
- **{depth}**: Layer depth (L1, L2, L3)
  - **L1**: Fundamentals (core concepts, essential vocabulary)
  - **L2**: Intermediate (advanced patterns, integration)
  - **L3**: Advanced (edge cases, failure modes, strategic thinking)
- **{semantic-concept}**: Kebab-case concept name (e.g., `hook-architecture`, `ralph-wiggum-loop`)

**Examples**:
- `3.1-L1-hook-architecture.md` - Lesson 3.1, fundamentals, hook architecture
- `3.17-L2-orchestration-patterns.md` - Lesson 3.17, intermediate, orchestration patterns

## YAML Frontmatter

Each file includes structured metadata:

```yaml
---
title: "Hook Architecture"
lesson: "3.1"
depth: "L1"
chapter: "3"
module: "General Agents"
concepts:
  - Hook System
  - Lifecycle Management
  - Callback Registration

  - TEACH Framework
  - What Goes Wrong Framework

  - Skipping Hook Registration

  - hooks
  - architecture
  - lifecycle

  - Event Loop
  - Async Patterns
---
```

### Required Fields
- `title` - Lesson title
- `lesson` - Lesson number (e.g., "3.1")
- `depth` - Layer depth ("L1", "L2", "L3")

### Optional Fields
- `chapter`, `module` - Organizational metadata
- `concepts` - List of key concepts
- `frameworks` - Frameworks introduced
- `anti_patterns` - Anti-patterns discussed
- `tags` - Tags for categorization
- `prerequisites` - Prerequisite concepts
- `related_concepts` - Related topics

## How Files Are Created

### Checkpoint Command
- Saves progress mid-lesson
- Creates new part file: `{X.Y}-L{depth}-{semantic-concept}.md`
- Does NOT merge with previous parts (versioned separately)

### Finish Command
- Creates final part file (if new content since last checkpoint)
- Reads ALL `{X.Y}-L*-*.md` files for the lesson
- Generates master INDEX.md referencing all parts

**Example progression**:
```
Session 1: Checkpoint → 3.1-L1-hook-architecture.md
Session 2: Checkpoint → 3.1-L2-custom-hooks.md
Session 3: Finish → 3.1-L3-advanced-patterns.md + INDEX.md updated
```
