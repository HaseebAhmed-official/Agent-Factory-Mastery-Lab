# Context Bridge

> **Purpose**: Session state preservation across conversations
> **Generated by**: Checkpoint and Finish commands
> **Format**: Cumulative markdown document

---

## What is a Context Bridge?

A **context bridge** is a living document that preserves your learning state across sessions. When you checkpoint or finish a lesson, Professor Agent updates the bridge with:

- Vocabulary learned
- Concepts covered
- Anti-patterns identified
- Quiz performance
- Checkpoint history
- Current progress

When you return, the bridge is reloaded so teaching continues seamlessly.

---

## File Naming

**Primary (always)**:
```
master-cumulative.md
```
Single living document for all sessions. Appended at each checkpoint. Never replaced.

**Backups (auto-created at each Checkpoint)**:
```
backup/master-cumulative-{YYYY-MM-DD}.md
```
Rolling 3-backup rotation. Protects against corruption.

**Snapshots (auto-created at each Checkpoint)**:
```
snapshots/lesson-{X.Y}-L{N}-{semantic-concept}-snapshot.md
```
Frozen point-in-time state used exclusively by the Rewind command. Read-only.

**Archive (auto-created when master > 300 lines)**:
```
archive/master-cumulative-archived-{YYYY-MM-DD}.md
```
Older bridge content moved here to keep active file lean.

## Structure

Each context bridge includes 16 sections:

### 1. Session Metadata
- Date, lesson, current checkpoint, session number

### 2. Lesson Overview
- Title, chapter, objectives, difficulty

### 3. Vocabulary Bank
- All terms defined (cumulative across checkpoints)

### 4. Knowledge Graph
- Tree structure of concepts covered

### 5. Core Concepts Summary
- Bullet-point list of key ideas

### 6. Completed Lessons
- List of fully finished lessons with dates

### 7. Frameworks Covered
- Structured approaches taught (e.g., TEACH cycle, OODA loop)

### 8. Anti-Patterns Identified
- Common mistakes and fixes

### 9. Hands-On Examples
- Code snippets and exercises completed

### 10. Edge Cases Explored
- Boundary conditions discussed

### 11. Strategic Insights
- When/why/trade-offs for each concept

### 12. Cross-References
- Links to related concepts in other lessons

### 13. Student Questions & Answers
- Key Q&A exchanges from session

### 14. Quiz Performance
- Scores, weak areas, improvement trends

### 15. Checkpoint History
- Table of all checkpoints with timestamps, concepts, files

### 16. Current Checkpoint State
- Active layer, last checkpoint time, context window status

## How It Works

### Initial Creation
1. User starts lesson (e.g., "Teach me 3.1")
2. Professor Agent creates context bridge: `session-01-lesson-3.1-origin-story.md`
3. Bridge initialized with lesson metadata

### Checkpoint Update
1. User types `Checkpoint`
2. Professor Agent appends new content to bridge:
   - New vocabulary → Section 3
   - New concepts → Sections 4, 5
   - Checkpoint entry → Section 15
3. Bridge saved, session continues

### Finish Update
1. User types `Finish`
2. Professor Agent finalizes bridge:
   - Marks lesson complete → Section 6
   - Final checkpoint → Section 15
   - Quiz performance (if taken) → Section 14
3. Bridge archived

### Session Reload (Cold-Start Recovery)
1. User returns in new conversation (or after `/clear`)
2. **Resume Protocol runs automatically** — reads `master-cumulative.md` before greeting
3. Recovery banner displayed:
   - Lesson, layer, last checkpoint date
   - Concepts since last checkpoint (or un-checkpointed teaching log content)
   - Repair prompt if any Stage 4 failures detected
4. Teaching resumes from exact saved state — no manual reload needed

See `Knowledge_Vault/Protocols/resume-protocol.md` for the full cold-start workflow.

## Cumulative vs Per-Lesson

**Strategy**: ONE cumulative bridge per session

- **Appends** new content each checkpoint
- **Grows** throughout lesson
- **Reloaded** at session start
- **Archived** after lesson complete

**NOT** separate bridges per checkpoint (too fragmented)

## Benefits

### Context Preservation
- No information loss between sessions
- Seamless continuation of teaching
- Maintains learning momentum

### Progress Tracking
- Clear history of what's been covered
- Checkpoint timeline visible
- Quiz performance tracked

### Strategic Resumption
- Professor Agent knows exactly where you left off
- Can suggest next steps based on history
- Identifies weak areas for review

## Usage

### Reading
- Open latest bridge to review progress
- Check vocabulary bank for quick reference
- Review checkpoint history

### Exporting
- Bridges are markdown (portable)
- Can be exported to PDF/HTML
- Useful for study review

### Debugging
- If session feels disconnected, check bridge load
- Verify bridge exists and has recent checkpoint
- Use `Status` to see current state
