# Quiz Bank

> **Purpose**: Assessment questions for knowledge verification and exam preparation
> **Generated by**: Professor Agent and Review command
> **Format**: Markdown with structured questions

---

## What's in the Quiz Bank?

A comprehensive collection of assessment questions covering all lessons:

- **Comprehension checks**: Verify understanding after lessons
- **Application scenarios**: Test ability to use concepts
- **Edge case analysis**: Identify boundary condition handling
- **Strategic thinking**: Evaluate when/why/trade-offs
- **Failure mode analysis**: Diagnose what goes wrong
- **Comparison questions**: Compare/contrast concepts

---

## Question Types

### 1. Explain-Back Questions
**Goal**: Articulate concept in your own words

**Example**:
```markdown
**Q**: Explain the Ralph Wiggum Loop pattern and describe a scenario where it would be problematic.

**Rubric**:
- Defines loop pattern (2 points)
- Explains why problematic (2 points)
- Provides concrete scenario (1 point)

**Model Answer**: [Full explanation]
```

### 2. Application Questions
**Goal**: Apply concept to new situation

**Example**:
```markdown
**Q**: You need to build an agent that processes 1000 documents. How would you design the workflow to avoid timeouts?

**Rubric**:
- Identifies batching strategy (2 points)
- Mentions error handling (1 point)
- Considers progress tracking (1 point)
- Discusses trade-offs (1 point)

**Model Answer**: [Detailed solution]
```

### 3. Failure Analysis Questions
**Goal**: Diagnose what went wrong

**Example**:
```markdown
**Q**: This agent code produces incorrect results. Identify the bug and explain the fix.

[Code snippet]

**Rubric**:
- Correctly identifies bug (2 points)
- Explains why it's wrong (2 points)
- Proposes correct fix (1 point)

**Model Answer**: [Diagnosis + fix]
```

### 4. Compare/Contrast Questions
**Goal**: Understand distinctions between concepts

**Example**:
```markdown
**Q**: Compare hooks vs middleware in agent architecture. When would you use each?

**Rubric**:
- Defines both correctly (2 points)
- Identifies key differences (2 points)
- Provides use case guidance (1 point)

**Model Answer**: [Comparison table + guidance]
```

### 5. Edge Case Questions
**Goal**: Handle boundary conditions

**Example**:
```markdown
**Q**: What happens if a hook throws an error during agent execution? How should it be handled?

**Rubric**:
- Describes behavior (2 points)
- Proposes handling strategy (2 points)
- Considers cascading effects (1 point)

**Model Answer**: [Complete analysis]
```

### 6. Strategic Questions
**Goal**: Evaluate approach and trade-offs

**Example**:
```markdown
**Q**: You're designing an agent that needs high reliability but also fast response times. What trade-offs would you make?

**Rubric**:
- Identifies tension (1 point)
- Lists trade-off options (2 points)
- Recommends approach with justification (2 points)

**Model Answer**: [Strategic analysis]
```

## Using Quizzes

### Self-Assessment
1. Complete lesson with `Finish`
2. Wait 1 day (spacing effect)
3. Take quiz WITHOUT notes
4. Grade yourself honestly
5. Review missed questions

### AI-Guided Review
```
User: Review 3.1
Professor Agent: [Loads quiz, asks questions interactively, provides feedback]
```

### Exam Preparation
- Take all quizzes for chapter
- Focus on consistently weak areas
- Reattempt low-scoring quizzes
- Use cumulative chapter quiz

---

## Difficulty Progression

- **Post-Lesson Quiz**: Immediate comprehension (easier)
- **Chapter Quiz**: Integration across lessons (moderate)
- **Cumulative Quiz**: Cross-chapter synthesis (harder)
- **Mock Exam**: Full course coverage (hardest)

## Generation

## Best Practices

### When Taking Quizzes
- ✅ Take 1 day after lesson (spacing)
- ✅ No notes on first attempt
- ✅ Grade yourself honestly
- ✅ Review ALL questions (even correct ones)
- ❌ Don't peek at answers
- ❌ Don't skip rubric evaluation

### After Quizzes
- Review weak areas immediately
- Reattempt failed questions next day
- Track improvement over time
- Note patterns in mistakes
