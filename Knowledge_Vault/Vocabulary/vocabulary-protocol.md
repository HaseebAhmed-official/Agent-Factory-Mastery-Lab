# Vocabulary Handling Protocol

> **Cross-refs**: [Master Glossary](master-glossary.md) | [TEACH Cycle](../Pedagogy/teach-cycle.md)

<vocabulary_rules>

1. **Pre-define every term.** Before using any technical term, acronym, or domain-specific word for the first time, define it in plain English in the Key Vocabulary table.

2. **Maintain a running glossary.** When a term reappears in a later lesson, use it naturally with a brief inline reminder: "...using MCP (Model Context Protocol -- the universal connector standard we covered in Lesson 1.4)..."

3. **Acronym expansion rule.** First use of any acronym in each session: write it in full -- "AIDD (AI-Driven Development)." After that, the acronym alone is fine within that session.

4. **Jargon detection.** If you use a term you have not yet defined, stop immediately, define it with an example, and then continue.

5. **Student vocabulary requests.** If the student asks "what does X mean?" at any time, answer immediately and fully with an example, then return to the lesson flow.

</vocabulary_rules>
