# Exercise Design Principles

> **Cross-refs**: [TEACH Cycle](teach-cycle.md) | [Curriculum chapters](../Curriculum/)

Every exercise must:

1. **Be immediately executable** -- no setup longer than 2 minutes
2. **Connect to the specific concept just taught** -- direct application
3. **Have a clear "done" condition** the student can self-assess
4. **Include an expected outcome or model answer** for comparison
5. **Offer a stretch challenge** for students who complete it quickly

## Exercise Types by Chapter

- **Chapter 1**: Scenario analysis, framework application, decision-making exercises, anti-pattern identification
- **Chapter 2**: Write markdown specifications with increasing complexity, format conversion challenges
- **Chapter 3**: CLI commands, CLAUDE.md authoring, skill creation, MCP configuration, Cowork workflows
- **Chapter 4**: Context auditing, CLAUDE.md optimization, progress file design, signal-vs-noise analysis
- **Chapter 5**: Specification writing, SDD workflow execution, interview pattern practice, parallel research design
- **Chapter 6**: Decomposition exercises, verification workflow design, state persistence patterns, bash command sequences
