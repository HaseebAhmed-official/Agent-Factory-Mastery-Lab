# Visual Presentations

> **Purpose**: Interactive HTML slide presentations for visual learning
> **Generated by**: Finish command (Stage 4 in finish-synthesis.md)
> **Format**: Standalone HTML with embedded CSS/JS

---

## What's in a Presentation?

Each lesson generates **two types** of HTML presentations:

### 1. Master Navigation HTML
- Overview of entire lesson
- Cards for each checkpoint (L1, L2, L3...)
- Links to individual checkpoint presentations
- Checkpoint timeline visualization
- Full lesson summary

**File**: `session-{NN}-lesson-{X.Y}-{lesson-kebab-title}.html`

**Example**: `session-01-lesson-3.1-origin-story.html`

### 2. Individual Checkpoint HTMLs
- Focused presentation for specific layer (L1, L2, L3)
- Deep dive into concepts covered at that depth
- Slide-based format with navigation

**Files**: `session-{NN}-lesson-{X.Y}-L{N}-presentation.html`

**Examples**:
- `session-01-lesson-3.1-L1-presentation.html` (Fundamentals)
- `session-01-lesson-3.1-L2-presentation.html` (Intermediate)
- `session-01-lesson-3.1-L3-presentation.html` (Advanced)

## Presentation Features

### Interactive Navigation
- **Arrow keys**: Previous/Next slide
- **Click**: Advance slide
- **Swipe**: Touch-friendly navigation
- **Dot indicators**: Current slide position
- **Progress bar**: Visual completion tracking

### Visual Elements
- **Fullscreen slides**: Immersive presentation mode
- **Animated transitions**: Smooth fade-in effects
- **Hover reveals**: Interactive tooltips and expansions
- **Accordion sections**: Collapsible content blocks
- **Flip cards**: Vocabulary term → definition
- **CSS-drawn diagrams**: Clean visual explanations (no ASCII art)

### Professional Theme
- **Color scheme**: Off-white background, indigo/cyan accents
- **Typography**: Clean, readable fonts
- **Layout**: Balanced whitespace, focused content
- **Responsive**: Works on desktop, tablet, mobile

## Slide Types

### 1. Title Slide
- Lesson title
- Chapter/Module info
- Difficulty level
- Estimated time

### 2. Vocabulary Slides
- Flip cards (front: term, back: definition)
- Context examples
- Related terms

### 3. Concept Slides
- What/Why/How structure
- Visual diagrams
- Code examples with syntax highlighting

### 4. Framework Slides
- Structured patterns
- Decision trees
- Application guidance

### 5. Anti-Pattern Slides
- ❌ What goes wrong
- ✅ How to fix
- Side-by-side comparison

### 6. Hands-On Slides
- Executable examples
- Expected outcomes
- Try-it-yourself prompts

### 7. Summary Slide
- Key takeaways
- Concept connections
- Next steps

## Generation Process

Presentations are **automatically generated** during `Finish` command:

1. Professor Agent reads ALL checkpoint part files (`{X.Y}-L*-*.md`)
2. Extracts content from each layer (L1, L2, L3...)
3. Generates master navigation HTML
4. Generates individual checkpoint HTMLs
5. Saves all files to this directory
6. Reports file paths to user

**No manual creation needed.**

## How to Use

### Viewing
1. **Open in browser**: Double-click any `.html` file
2. **Navigate**: Use arrow keys or click to advance
3. **Full lesson**: Start with master navigation HTML
4. **Specific depth**: Open individual checkpoint HTML

### Presenting
- Perfect for screen sharing during study groups
- Use fullscreen mode (F11 in most browsers)
- Navigate with arrow keys or presenter remote

### Reviewing
- Quick visual refresh before quizzes
- Scan slides for key concepts
- Use as study guide

### Sharing
- Standalone HTML (no external dependencies)
- Send file directly via email/chat
- Host on web server or GitHub Pages

## Customization

### Editing Presentations
- Presentations are regenerated on each `Finish`
- Edit source checkpoint files, then run `Finish` again
- Manual HTML edits will be overwritten

### Styling
- Embedded CSS in `<style>` tags
- Modify template in `Knowledge_Vault/Protocols/finish-synthesis.md` (STAGE 4)
- Or edit generated HTML for one-off changes

## Troubleshooting

### Presentation not loading
- Ensure JavaScript is enabled in browser
- Check console for errors (F12 developer tools)
- Verify file is complete (not truncated)

### Slides not advancing
- Use arrow keys (← →) or click
- Check if modal/overlay is blocking
- Refresh page (F5)

### Missing content
- Verify all checkpoint files exist for lesson
- Re-run `Finish` command to regenerate
- Check that checkpoint files have content
