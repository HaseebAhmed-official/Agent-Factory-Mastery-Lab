Principle 2

Parallel Sessions

"The single biggest productivity unlock" — Claude Code Team

Single Session

Task A → Task B → Task C in one window. Context fills, cross-contaminates, degrades.

Context: 90% polluted
VS

Parallel Sessions

3 terminals, 3 worktrees, 3 focused sessions. Each task gets full context budget.

Each: 20-30% focused
For Developers
git worktree add ../auth feature/auth
git worktree add ../bugfix bugfix/api
cd ../auth && claude
For Knowledge Workers
Multiple browser tabs or Cowork workspaces. Color-code by project. Name sessions with /rename.

What Goes Wrong

Anti-Patterns

Multiple unrelated tasks in one session. Context cross-contaminates and degrades.

Fix: /clear between unrelated tasks

Corrected Claude 2+ times, still wrong. Failed approaches poison the context window.

Fix: After 2 failures → /clear + rewrite prompt with learnings

800-line CLAUDE.md where important rules get buried. Claude ignores instructions.

Fix: Ruthlessly prune. If Claude already does it right, delete the rule.

Plausible-looking output that doesn't handle edge cases. No verification infrastructure.

Fix: Always provide verification methods (tests, hooks, subagents)

Unscoped "investigate" fills context with file reads. Budget exhausted before real work begins.

Fix: Scope narrowly OR use subagents for isolated exploration
