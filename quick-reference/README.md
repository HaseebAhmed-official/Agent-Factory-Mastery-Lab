# Quick Reference Cheatsheets

> **Purpose**: Condensed 2-3 page summaries of lessons for rapid lookup
> **Generated by**: Finish command (Stage 5 in finish-synthesis.md)
> **Format**: Markdown (GitHub-flavored)

---

## What's in a Cheatsheet?

Each cheatsheet provides:

1. **At a Glance** - One-sentence concept summary, difficulty, estimated time
2. **Essential Vocabulary** - All terms with one-line definitions
3. **Key Frameworks** - Compact formulas and decision trees
4. **Common Pitfalls** - Top anti-patterns with fixes
5. **Quick Commands** - Code snippets and procedures
6. **Where to Learn More** - Links to full notes, HTML presentations, exercises

---

## File Naming Convention

`lesson-{X.Y}-cheatsheet.md`

**Examples**:
- `lesson-3.1-cheatsheet.md` (Lesson 3.1: Claude Code Origin Story)
- `lesson-3.17-cheatsheet.md` (Lesson 3.17: Ralph Wiggum Loop)

---

## How to Use

### Quick Lookup
Open the cheatsheet for rapid reference during:
- Coding/implementation
- Exam prep
- Quick refresher before next lesson

### Study Workflow
1. Complete lesson (use Finish command)
2. Review HTML presentation (visual learning)
3. Study full notes (deep learning)
4. **Use cheatsheet** (rapid recall test)
5. Review flashcards (spaced repetition)

---

## Generation Process

Cheatsheets are **automatically generated** when you use the `Finish` command:

1. System reads ALL checkpoint part files for the lesson
2. Extracts vocabulary, frameworks, decision trees, anti-patterns
3. Condenses to 2-3 pages (essential information only)
4. Saves to this directory

You do NOT need to manually create cheatsheets.

---

## Export Options

Convert to other formats using export scripts:

```bash
# Convert to PDF
python scripts/export-to-pdf.sh quick-reference/lesson-3.1-cheatsheet.md

# Convert to standalone HTML
python scripts/export-to-html.sh quick-reference/lesson-3.1-cheatsheet.md
```

## Example Structure

```markdown
# Lesson 3.1 Quick Reference: Hook Architecture

## At a Glance
- Core Concept: Callback functions that execute at lifecycle points
- When to Use: Extend agent behavior without modifying core
- Difficulty: Intermediate
- Estimated Time: 25 min

## Essential Vocabulary
| Term | Definition |
|------|-----------|
| Hook | Callback function registered to execute at specific points |
| Lifecycle | Sequence of states an agent transitions through |

## Key Frameworks
1. **OODA Loop**: Observe → Orient → Decide → Act

## Common Pitfalls
| ❌ Anti-Pattern | ✅ Fix |
|----------------|--------|
| Conditional hook registration | Always register hooks in same order |

## Quick Commands
```bash
# Register a hook
agent.registerHook('lifecycle', callback)
```

## Where to Learn More
- Full notes: `revision-notes/.../3.1-L*-*.md`
- Visual: `visual-presentations/session-NN-lesson-3.1-*.html`
```

## Maintenance

Cheatsheets are regenerated each time you use the Finish command for a lesson.

If you update lesson notes and want to refresh the cheatsheet:
1. Use `Finish` command again (will overwrite existing cheatsheet)
2. OR manually edit the cheatsheet markdown file
