# Exercises

> **Purpose**: Hands-on practice exercises for active learning
> **Generated by**: Professor Agent during lessons
> **Format**: Markdown with embedded code examples

---

## What Are Exercises?

Exercises are **immediately executable** practice problems that bridge theory to practice. Each exercise includes:

- **Setup instructions**: Prerequisites and environment setup
- **Task description**: Clear objective and acceptance criteria
- **Expected outcome**: What success looks like
- **Solution approach**: Hints without spoiling the answer
- **Verification**: How to check if you got it right

---

## Exercise Types

### 1. Concept Application
**Goal**: Apply learned concept to new scenario

**Example**:
```markdown
# Exercise: Hook Registration Order

**Concept**: Hook lifecycle and registration timing

**Task**: Given this agent code, identify the bug and fix it:
[code snippet]

**Expected**: Agent should execute hooks in correct order

**Hint**: Check when hooks are registered vs when they execute
```

### 2. Build From Scratch
**Goal**: Implement concept from first principles

**Example**:
```markdown
# Exercise: Custom Hook System

**Task**: Build a minimal hook system with:
- Register hook by name
- Trigger hook with data
- Support multiple callbacks per hook

**Requirements**:
- Use plain JavaScript/Python
- No external libraries
- Pass provided test cases

[Test cases provided]
```

### 3. Debug Challenge
**Goal**: Identify and fix broken code

**Example**:
```markdown
# Exercise: Agent Loop Deadlock

**Scenario**: This agent hangs after 3 iterations

[Broken code]

**Task**: Debug and fix the deadlock

**Hint**: Check async/await patterns
```

### 4. Edge Case Exploration
**Goal**: Handle boundary conditions

**Example**:
```markdown
# Exercise: Error Handling Edge Cases

**Task**: Make this agent robust against:
- Network timeouts
- Malformed responses
- Rate limiting
- Partial failures

[Base code]

**Requirements**: Agent should gracefully handle all cases
```

### 5. Optimization Challenge
**Goal**: Improve performance or design

**Example**:
```markdown
# Exercise: Reduce Token Usage

**Current**: This prompt uses 500 tokens
**Goal**: Reduce to <300 tokens without losing functionality

[Verbose prompt]

**Constraints**: Maintain all requirements
```

## File Naming

**Format**: `{X.Y}-{type}-{name}.md`

**Examples**:
- `3.1-concept-hook-registration.md`
- `3.17-debug-loop-infinite.md`
- `3.22-build-custom-workflow.md`

**Types**:
- `concept` - Apply concept to scenario
- `build` - Build from scratch
- `debug` - Fix broken code
- `edge` - Handle edge cases
- `optimize` - Improve design/performance

## Usage

### Solo Practice
1. Read task description
2. Attempt without looking at hints
3. Use hints if stuck
4. Verify your solution
5. Compare with provided solution

### Study Groups
- Work in pairs/groups
- Discuss approach before coding
- Compare solutions
- Learn from different approaches

### Exam Prep
- Practice exercises from weak areas
- Time yourself
- Review solutions afterward

---

## Difficulty Levels

- **Beginner**: Apply single concept directly (10-15 min)
- **Intermediate**: Combine multiple concepts (20-30 min)
- **Advanced**: Design solution with trade-offs (30-45 min)
- **Expert**: Open-ended, multiple valid approaches (1+ hour)

---

## Best Practices

### When Attempting
- ✅ Read requirements carefully
- ✅ Plan approach before coding
- ✅ Test incrementally
- ✅ Check edge cases
- ❌ Don't peek at solution immediately
- ❌ Don't skip verification

### After Completing
- Review solution (compare approaches)
- Note what you learned
- Identify patterns for future use
- Try extension challenges
